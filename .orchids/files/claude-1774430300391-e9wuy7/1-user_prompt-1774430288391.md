Build authentication system, database integration, and notification system for an existing SaaS app called "SalonOS".

Use:
- Supabase for database and authentication
- Firebase for push notifications

---

AUTHENTICATION SYSTEM:

Create 3 types of login with role-based access:

1. Customer Login:
- Mobile number login with OTP verification
- Auto create user account
- Role = "customer"

2. Salon Owner Login:
- Email + password login
- Ability to create a new salon or join existing salon
- Role = "owner"

3. Admin Login:
- Email + password login
- Role = "admin"
- Full system access

---

USER ROLE LOGIC:

- Each user must have a role field: customer / owner / admin
- Each user must have tenant_id (salon id)
- Admin can access all data
- Owners can access only their salon data
- Customers can access only their own bookings

---

DATABASE (SUPABASE):

Create or connect these tables:

1. salons
- id (UUID)
- name
- owner_name
- phone
- created_at

2. users
- id (linked with auth user)
- name
- phone
- role (customer / owner / admin)
- tenant_id (salon id)

3. bookings
- id
- user_id
- service_id
- booking_time
- status
- tenant_id

4. queue
- id
- user_id
- position
- status
- tenant_id

5. notifications
- id
- user_id
- message
- read_status
- tenant_id

6. subscriptions
- id
- tenant_id
- plan
- expiry_date

---

MULTI-TENANT DATA RULE:

- Every table must include tenant_id
- All queries must filter using tenant_id
- Ensure strict data isolation between salons

---

ROW LEVEL SECURITY (IMPORTANT):

- Enable Row Level Security (RLS) in Supabase
- Only allow users to access their own tenant data
- Admin can bypass all restrictions

---
